A personal website and portfolio built with Quarto, showcasing professional work, research, projects, and interests. The site combines LaTeX typesetting with modern web technologies to create a comprehensive academic and professional presence.

## Overview

This repository contains the source code for a personal website that serves as both a digital resume and portfolio. Built using Quarto, it leverages the power of LaTeX for beautiful document formatting while maintaining modern web accessibility and responsiveness.

## Features

- **Professional Portfolio**: Comprehensive showcase of projects and research
- **Academic Focus**: Dedicated sections for research, teaching, and presentations
- **Personal Touch**: Musings and passions sections for a well-rounded profile
- **Responsive Design**: Custom CSS for optimal viewing across devices
- **Static Site**: Fast loading with GitHub Pages compatibility

## Structure

The website is organized into the following main sections:

- **About**: Personal and professional background
- **Projects**: Portfolio of completed work and ongoing initiatives
- **Research**: Academic research and publications
- **Teaching**: Educational experience and materials
- **Talks & Presentations**: Speaking engagements and conference presentations
- **Passions**: Personal interests and hobbies
- **Musings**: Blog-style thoughts and reflections

## Local Development

To work with this site locally:

1. Install [Quarto](https://quarto.org/docs/get-started/)
2. Clone this repository
3. Run `quarto preview` to start a local development server
4. Make changes to `.qmd` files and see live updates

## Deployment

The site is configured for GitHub Pages deployment with the `.nojekyll` file and appropriate Quarto configuration. Changes to the main branch will automatically trigger a rebuild and deployment.

## Customization

- Modify `_quarto.yml` for site-wide configuration
- Update CSS files in the `css/` directory for styling changes
- Add or modify content in the respective `.qmd` files
- Place additional assets in the `files/` directory
